# AWSomeShop 需求文档

## 文档信息

- **项目名称**: AWSomeShop 内部员工福利电商系统
- **文档版本**: 1.0
- **创建日期**: 2026-01-14
- **文档状态**: 待审批
- **项目类型**: MVP（最小可行产品）

## 1. 项目概述

### 1.1 项目背景
构建一个名为 AWSomeShop 的内部员工福利电商网站，旨在通过最小可行产品（MVP）验证员工积分兑换系统的商业模式。

### 1.2 项目目标
- 为员工提供使用"AWSome积分"浏览和兑换预选产品的平台
- 为管理员提供产品管理和积分管理的后台系统
- 验证积分兑换系统的可行性和员工接受度

### 1.3 项目范围
本项目为 MVP 版本，聚焦核心功能：
- 员工端：产品浏览、积分查询、产品兑换、订单管理
- 管理员端：产品管理、员工管理、积分管理、订单审核

### 1.4 目标用户
- **员工**: 约 1000 名内部员工
- **管理员**: 负责系统管理和运营的管理人员

## 2. 用户角色定义

### 2.1 员工（Employee）
**角色描述**: 使用积分兑换产品的内部员工

**主要职责**:
- 浏览和查看产品信息
- 使用积分兑换产品
- 查看个人积分余额和变动记录
- 管理个人兑换订单
- 修改个人密码

**访问权限**:
- 查看已上架和已下架的产品
- 查看个人信息（不可修改）
- 查看个人积分日志
- 查看和管理个人兑换订单

### 2.2 管理员（Administrator）
**角色描述**: 负责系统管理和运营的管理人员

**主要职责**:
- 管理产品信息（增删改查）
- 管理员工账号（增删改查）
- 管理其他管理员账号（增删改查）
- 发放和扣除员工积分
- 审核员工兑换订单
- 查看所有积分日志
- 查看系统概况

**访问权限**:
- 所有管理员拥有相同的全部管理权限

## 3. 功能需求

### 3.1 用户认证与授权

#### FR-AUTH-001: 员工登录
**需求描述**: 员工使用工号和密码登录系统

**验收标准（EARS）**:
- WHEN 员工输入正确的工号和密码 THEN 系统应允许登录并跳转到产品列表页
- WHEN 员工输入错误的工号或密码 THEN 系统应显示错误提示信息
- WHEN 员工账号状态为"离职" THEN 系统应拒绝登录并提示账号已禁用
- WHEN 员工成功登录 THEN 系统应创建会话，有效期为 24 小时

**优先级**: 高
**依赖**: 无

#### FR-AUTH-002: 管理员登录
**需求描述**: 管理员使用工号和密码登录系统

**验收标准（EARS）**:
- WHEN 管理员输入正确的工号和密码 THEN 系统应允许登录并跳转到系统概况页
- WHEN 管理员输入错误的工号或密码 THEN 系统应显示错误提示信息
- WHEN 管理员成功登录 THEN 系统应创建会话，有效期为 24 小时

**优先级**: 高
**依赖**: 无

#### FR-AUTH-003: 会话管理
**需求描述**: 系统管理用户登录会话

**验收标准（EARS）**:
- WHEN 用户登录成功 THEN 系统应创建会话，有效期为 24 小时
- WHEN 会话过期 THEN 系统应要求用户重新登录
- WHEN 用户主动登出 THEN 系统应立即销毁会话

**优先级**: 高
**依赖**: FR-AUTH-001, FR-AUTH-002

#### FR-AUTH-004: 员工修改密码
**需求描述**: 员工可以修改自己的登录密码

**验收标准（EARS）**:
- WHEN 员工输入正确的旧密码和新密码 THEN 系统应更新密码并提示成功
- WHEN 员工输入错误的旧密码 THEN 系统应拒绝修改并提示旧密码错误
- WHERE 密码无复杂度要求

**优先级**: 中
**依赖**: FR-AUTH-001

#### FR-AUTH-005: 管理员修改密码
**需求描述**: 管理员可以修改自己的登录密码

**验收标准（EARS）**:
- WHEN 管理员输入正确的旧密码和新密码 THEN 系统应更新密码并提示成功
- WHEN 管理员输入错误的旧密码 THEN 系统应拒绝修改并提示旧密码错误

**优先级**: 中
**依赖**: FR-AUTH-002

#### FR-AUTH-006: 管理员重置员工密码
**需求描述**: 管理员可以重置员工密码

**验收标准（EARS）**:
- WHEN 管理员重置密码成功 THEN 系统应提示成功并记录操作

**优先级**: 中
**依赖**: FR-AUTH-002

### 3.2 员工管理

#### FR-EMP-001: 创建员工账号
**需求描述**: 管理员可以创建员工账号

**验收标准（EARS）**:
- WHEN 管理员创建员工账号 THEN 系统应要求填写：姓名、工号、部门、岗位、邮箱、手机号、入职时间、初始密码、状态
- WHEN 员工账号创建成功 THEN 系统应自动为该员工发放 1000 积分
- WHERE 工号长度不超过 50 个字符
- WHERE 员工状态包括：在职、离职、试用期
- WHERE 不同状态的员工权限无差异

**优先级**: 高
**依赖**: FR-AUTH-002

#### FR-EMP-002: 编辑员工信息
**需求描述**: 管理员可以编辑员工信息

**验收标准（EARS）**:
- WHEN 管理员编辑员工信息 THEN 系统应允许修改除工号外的所有字段
- WHERE 工号不可修改

**优先级**: 中
**依赖**: FR-EMP-001

#### FR-EMP-003: 禁用员工账号
**需求描述**: 管理员可以禁用员工账号

**验收标准（EARS）**:
- WHEN 管理员将员工状态设置为"离职" THEN 系统应禁用该员工账号
- WHEN 员工账号被禁用 THEN 该员工无法登录系统
- WHERE 员工剩余积分不做处理

**优先级**: 中
**依赖**: FR-EMP-001

#### FR-EMP-004: 查看员工列表
**需求描述**: 管理员可以查看员工列表

**验收标准（EARS）**:
- WHEN 管理员查看员工列表 THEN 系统应显示所有员工信息
- WHEN 管理员使用筛选功能 THEN 系统应支持按姓名、工号、部门、岗位筛选
- WHERE 列表采用分页显示，每页 50 条记录

**优先级**: 中
**依赖**: FR-EMP-001

#### FR-EMP-005: 员工个人中心
**需求描述**: 员工可以查看个人信息和相关数据

**验收标准（EARS）**:
- WHEN 员工访问个人中心 THEN 系统应显示导航：个人信息、积分余额、兑换历史、修改密码入口
- WHEN 员工查看个人信息 THEN 系统应显示所有个人字段信息
- WHERE 员工只能查看个人信息，不能修改

**优先级**: 中
**依赖**: FR-AUTH-001

### 3.3 管理员管理

#### FR-ADMIN-001: 创建管理员账号
**需求描述**: 管理员可以创建其他管理员账号

**验收标准（EARS）**:
- WHEN 管理员创建管理员账号 THEN 系统应要求填写：姓名、工号、部门、岗位、邮箱、手机号、入职时间、初始密码、状态

**优先级**: 高
**依赖**: FR-AUTH-002

#### FR-ADMIN-002: 系统初始化
**需求描述**: 系统首次部署时自动初始化默认数据

**验收标准（EARS）**:
- WHEN 系统首次部署 THEN 系统应创建一个 admin 管理员账号
- WHEN 系统首次部署 THEN 系统应初始化示例产品：9 个示例产品

**优先级**: 高
**依赖**: 无

### 3.4 产品管理

#### FR-PROD-001: 创建产品
**需求描述**: 管理员可以创建产品

**验收标准（EARS）**:
- WHEN 管理员创建产品 THEN 系统应要求填写：名称、描述、图片、所需积分
- WHERE 产品名称长度不超过 100 个字符
- WHERE 产品描述长度不超过 2000 个字符
- WHERE 产品图片大小不超过 1MB，格式不限
- WHERE 新创建的产品默认状态为"下架"

**优先级**: 高
**依赖**: FR-AUTH-002

#### FR-PROD-002: 编辑产品
**需求描述**: 管理员可以编辑产品信息

**验收标准（EARS）**:
- WHEN 管理员编辑产品 THEN 系统应允许修改所有产品字段
- WHERE 不需要保存修改历史

**优先级**: 中
**依赖**: FR-PROD-001

#### FR-PROD-003: 删除产品
**需求描述**: 管理员可以删除产品

**验收标准（EARS）**:
- WHEN 管理员删除产品 THEN 系统应执行软删除
- WHEN 产品已有兑换记录 THEN 系统应保留产品数据以维护订单完整性
- WHERE 已删除的产品不在产品列表中显示

**优先级**: 中
**依赖**: FR-PROD-001

#### FR-PROD-004: 产品上架/下架
**需求描述**: 管理员可以上架或下架产品

**验收标准（EARS）**:
- WHEN 管理员上架产品 THEN 员工可以兑换该产品
- WHEN 管理员下架产品 THEN 员工可以查看但无法兑换该产品
- WHEN 员工查看下架产品 THEN 系统应标记"已下架"

**优先级**: 高
**依赖**: FR-PROD-001

#### FR-PROD-005: 产品图片管理
**需求描述**: 管理员可以管理产品图片

**验收标准（EARS）**:
- WHERE 每个产品只能上传一张图片
- WHEN 管理员上传新图片 THEN 系统应支持删除和替换现有图片

**优先级**: 中
**依赖**: FR-PROD-001

#### FR-PROD-006: 查看产品列表（管理员）
**需求描述**: 管理员可以查看产品列表

**验收标准（EARS）**:
- WHEN 管理员查看产品列表 THEN 系统应显示所有产品（包括已删除的）
- WHERE 产品列表按创建时间倒序排列
- WHERE 列表采用分页显示，每页 50 条记录

**优先级**: 中
**依赖**: FR-PROD-001

### 3.5 员工端产品浏览

#### FR-PROD-VIEW-001: 查看产品列表（员工）
**需求描述**: 员工可以浏览产品列表

**验收标准（EARS）**:
- WHEN 员工登录后 THEN 系统应显示产品列表作为首页
- WHEN 员工查看产品列表 THEN 系统应显示所有产品（包括已上架和已下架）
- WHERE 产品列表按创建时间倒序排列
- WHERE 列表采用分页显示，每页 50 条记录
- WHERE 不支持产品搜索功能
- WHERE 不支持筛选和排序功能

**优先级**: 高
**依赖**: FR-AUTH-001, FR-PROD-001

#### FR-PROD-VIEW-002: 查看产品详情（员工）
**需求描述**: 员工可以查看产品详细信息

**验收标准（EARS）**:
- WHEN 员工点击产品 THEN 系统应显示产品详情页
- WHEN 员工查看产品详情 THEN 系统应显示所有产品字段：名称、描述、图片、所需积分、上架状态
- WHEN 产品已下架 THEN 系统应标记"已下架"并禁用兑换按钮
- WHEN 员工积分不足 THEN 系统应显示"积分不足"提示
- WHERE 不显示"已兑换人数"或"热门程度"信息

**优先级**: 高
**依赖**: FR-PROD-VIEW-001

### 3.6 积分管理

#### FR-POINT-001: 入职积分发放
**需求描述**: 系统在创建员工账号时自动发放入职积分

**验收标准（EARS）**:
- WHEN 管理员创建员工账号 THEN 系统应自动为该员工发放 1000 积分
- WHEN 积分发放成功 THEN 系统应记录积分日志，变动类型为"入职发放"

**优先级**: 高
**依赖**: FR-EMP-001

#### FR-POINT-002: 管理员发放积分
**需求描述**: 管理员可以为员工发放积分

**验收标准（EARS）**:
- WHEN 管理员发放积分 THEN 系统应要求填写：目标员工、积分数量、发放原因
- WHEN 积分发放成功 THEN 系统应更新员工积分余额并记录积分日志
- WHERE 积分数量无上限
- WHERE 积分最小单位为 1

**优先级**: 高
**依赖**: FR-AUTH-002

#### FR-POINT-003: 批量发放积分
**需求描述**: 管理员可以批量发放积分

**验收标准（EARS）**:
- WHEN 管理员批量发放积分 THEN 系统应支持选择目标员工：全选所有员工、按部门选择、按岗位选择、手动勾选
- WHEN 管理员批量发放积分 THEN 系统应要求填写：积分数量、发放原因
- WHEN 批量发放成功 THEN 系统应为每个目标员工更新积分余额并记录积分日志
- WHERE 不需要预览功能

**优先级**: 中
**依赖**: FR-POINT-002

#### FR-POINT-004: 管理员扣除积分
**需求描述**: 管理员可以扣除员工积分

**验收标准（EARS）**:
- WHEN 管理员扣除积分 THEN 系统应要求填写：目标员工、积分数量、扣除原因
- WHEN 积分扣除成功 THEN 系统应更新员工积分余额并记录积分日志
- WHERE 不需要员工确认或审批

**优先级**: 中
**依赖**: FR-POINT-002

#### FR-POINT-005: 查看积分余额（员工）
**需求描述**: 员工可以查看自己的积分余额和明细

**验收标准（EARS）**:
- WHEN 员工查看积分余额 THEN 系统应显示当前积分总额
- WHEN 员工查看积分明细 THEN 系统应显示积分变动记录（获得/消费记录）
- WHERE 积分明细按时间倒序排列
- WHERE 积分明细采用分页显示，每页 50 条记录

**优先级**: 高
**依赖**: FR-AUTH-001

#### FR-POINT-006: 积分日志记录
**需求描述**: 系统记录所有积分变动操作

**验收标准（EARS）**:
- WHEN 发生积分变动 THEN 系统应记录：操作时间、操作人、员工、积分变动数量、变动原因、变动类型
- WHERE 变动类型包括：入职发放、管理员发放、管理员扣除、兑换消费、订单取消退回、订单拒绝退回

**优先级**: 高
**依赖**: 所有涉及积分变动的功能

#### FR-POINT-007: 查看积分日志（管理员）
**需求描述**: 管理员可以查看所有积分日志

**验收标准（EARS）**:
- WHEN 管理员查看积分日志 THEN 系统应显示所有员工的积分变动记录
- WHERE 积分日志按时间倒序排列
- WHERE 积分日志采用分页显示，每页 50 条记录
- WHERE 不支持按时间范围筛选

**优先级**: 中
**依赖**: FR-POINT-006

#### FR-POINT-008: 查看积分日志（员工）
**需求描述**: 员工可以查看自己的积分日志

**验收标准（EARS）**:
- WHEN 员工查看积分日志 THEN 系统应只显示该员工的积分变动记录
- WHERE 积分日志按时间倒序排列
- WHERE 积分日志采用分页显示，每页 50 条记录

**优先级**: 中
**依赖**: FR-POINT-006

### 3.7 产品兑换

#### FR-REDEEM-001: 提交兑换申请
**需求描述**: 员工可以使用积分兑换产品

**验收标准（EARS）**:
- WHEN 员工点击兑换按钮 THEN 系统应验证产品是否已上架
- WHEN 产品已下架 THEN 系统应拒绝兑换并提示"产品已下架"
- WHEN 员工积分不足 THEN 系统应拒绝兑换并提示"积分不足"
- WHEN 员工积分充足且产品已上架 THEN 系统应立即扣除积分并创建兑换订单
- WHEN 兑换订单创建成功 THEN 系统应生成订单号（时间戳+随机数）
- WHEN 兑换订单创建成功 THEN 订单状态为"待核销"
- WHERE 员工一次只能兑换一个产品
- WHERE 同一产品可以多次兑换，无次数限制
- WHERE 不需要填写收货地址等信息

**优先级**: 高
**依赖**: FR-AUTH-001, FR-PROD-VIEW-002, FR-POINT-005

#### FR-REDEEM-002: 查看兑换订单（员工）
**需求描述**: 员工可以查看自己的兑换订单

**验收标准（EARS）**:
- WHEN 员工查看兑换订单 THEN 系统应显示所有订单信息：订单号、产品信息、所需积分、订单状态、申请时间、审核时间、审核人、拒绝原因/核销备注
- WHERE 订单列表按创建时间倒序排列
- WHERE 订单列表采用分页显示，每页 50 条记录

**优先级**: 高
**依赖**: FR-REDEEM-001

#### FR-REDEEM-003: 取消兑换订单
**需求描述**: 员工可以取消待核销状态的兑换订单

**验收标准（EARS）**:
- WHEN 订单状态为"待核销" THEN 员工可以取消订单
- WHEN 订单状态不是"待核销" THEN 系统应拒绝取消操作
- WHEN 员工取消订单成功 THEN 系统应立即退回积分并更新订单状态为"已取消"
- WHEN 积分退回成功 THEN 系统应记录积分日志，变动类型为"订单取消退回"

**优先级**: 高
**依赖**: FR-REDEEM-001

### 3.8 订单审核

#### FR-ORDER-001: 查看待审核订单列表
**需求描述**: 管理员可以查看待审核订单列表

**验收标准（EARS）**:
- WHEN 管理员查看订单列表 THEN 系统应显示所有兑换订单
- WHEN 管理员使用状态筛选 THEN 系统应支持按订单状态筛选：待核销、已核销、已取消、已拒绝
- WHERE 订单列表按创建时间倒序排列
- WHERE 订单列表采用分页显示，每页 50 条记录

**优先级**: 高
**依赖**: FR-AUTH-002

#### FR-ORDER-002: 核销订单
**需求描述**: 管理员可以核销订单

**验收标准（EARS）**:
- WHEN 管理员核销订单 THEN 系统应要求填写核销备注（选填）
- WHEN 核销成功 THEN 系统应更新订单状态为"已核销"并记录审核时间和审核人
- WHERE 核销备注用于记录特殊情况，如：代领、产品有破损等

**优先级**: 高
**依赖**: FR-ORDER-001

#### FR-ORDER-003: 拒绝订单
**需求描述**: 管理员可以拒绝订单

**验收标准（EARS）**:
- WHEN 管理员拒绝订单 THEN 系统应要求填写拒绝原因（必填）
- WHEN 拒绝成功 THEN 系统应立即退回积分并更新订单状态为"已拒绝"
- WHEN 积分退回成功 THEN 系统应记录积分日志，变动类型为"订单拒绝退回"
- WHEN 拒绝成功 THEN 系统应记录审核时间和审核人

**优先级**: 高
**依赖**: FR-ORDER-001

#### FR-ORDER-004: 订单状态管理
**需求描述**: 系统管理订单的状态流转

**验收标准（EARS）**:
- WHERE 订单状态包括：待核销、已核销、已取消、已拒绝
- WHEN 订单创建 THEN 初始状态为"待核销"
- WHEN 管理员核销 THEN 状态变更为"已核销"
- WHEN 管理员拒绝 THEN 状态变更为"已拒绝"
- WHEN 员工取消 THEN 状态变更为"已取消"
- WHERE 订单历史永久保留

**优先级**: 高
**依赖**: FR-REDEEM-001

## 6. 业务规则

### 6.1 积分规则
1. 新员工入职时自动获得 1000 积分
2. 积分最小单位为 1，不支持小数
3. 积分无有效期，永久有效
4. 积分无上限
5. 员工离职后积分不做处理，保留在账户中

### 6.2 兑换规则
1. 员工一次只能兑换一个产品
2. 同一产品可以多次兑换，无次数限制
3. 只有已上架的产品可以兑换
4. 兑换时立即扣除积分
5. 订单取消或被拒绝时立即退回积分

### 6.3 订单规则
1. 订单创建后状态为"待核销"
2. 只有"待核销"状态的订单可以被员工取消
3. 订单历史永久保留
4. 订单号格式：时间戳+随机数

### 6.4 权限规则
1. 所有管理员拥有相同的全部管理权限
2. 管理员可以创建和管理其他管理员账号

### 6.5 产品规则
1. 产品删除使用软删除
2. 已有兑换记录的产品数据必须保留
3. 新创建的产品默认为"下架"状态
4. 下架的产品员工可以查看但无法兑换
