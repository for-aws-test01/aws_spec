# AWSomeShop 项目规格说明文档

## 项目概述

AWSomeShop 是一个内部员工福利电商系统，允许员工使用"AWSome积分"兑换预选产品。本仓库包含完整的项目规格说明文档。

## 文档结构

### 📋 规划文档
- `requirements_plan.md` - 需求收集规划
- `design_plan.md` - 设计规划
- `tasks_plan.md` - 任务分解规划

### 📄 规格文档（.kiro/specs/）

#### 1. 需求文档
- `requirements.md` - 完整的需求规格说明
  - 使用 EARS 模式编写验收标准
  - 遵循 INCOSE 质量规则
  - 包含 46 个功能需求和非功能需求

#### 2. 设计文档
- `design.md` - 详细的技术设计方案
  - 系统架构设计
  - 数据库设计（6张表）
  - API 接口规范（40+ 接口）
  - 前端设计
  - 安全和性能设计
  - 部署方案

#### 3. 架构文档
- `architecture.md` - C4 模型架构图（Mermaid）
  - Level 1: 系统上下文图
  - Level 2: 容器图
  - Level 3: 组件图（后端 + 前端）
  - 数据流图
  - 部署架构图
  - 数据库 ER 图

#### 4. 任务文档
- `tasks.md` - 详细的任务分解（85个任务）
  - 14 个开发阶段
  - 每个任务包含验收标准
  - 依赖关系和工作量估算
  - 预估总工期：28-30 周

- `tasks-parallel.md` - 并行任务执行计划
  - 优化后的任务分组
  - 标注可并行执行的任务
  - 前后端并行开发策略
  - **优化后工期：13 周（节省 54%）**

## 技术栈

### 后端
- **语言**: Go 1.21+
- **框架**: Gin
- **ORM**: GORM
- **数据库**: MySQL 8.0
- **认证**: JWT
- **密码**: bcrypt

### 前端
- **语言**: JavaScript (ES6+)
- **UI**: Ant Design CSS
- **构建**: Webpack 5
- **服务器**: Express
- **路由**: History API

### 开发工具
- **版本控制**: Git
- **容器化**: Docker Compose
- **测试**: Go testing + testify
- **API 文档**: Swagger

## 核心功能

### 员工端
- 浏览和兑换产品
- 查看积分余额和明细
- 管理兑换订单
- 修改个人密码

### 管理员端
- 员工管理（创建、编辑、重置密码）
- 产品管理（CRUD、上下架）
- 积分管理（发放、扣除、批量发放）
- 订单审核（核销、拒绝）
- 系统概况统计
